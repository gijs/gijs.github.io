Objective-C
-----------
* [Helios](http://helios.io) is an open-source framework that provides essential backend services for iOS apps. Features data synchronization, push notifications, in-app purchases and passbook integration. Based on Rails.

* [Try Objective-C](http://tryobjectivec.codeschool.com) is an online course for learning that language and iOS programming.

Python
------

* [Dataset](http://dataset.readthedocs.org/en/latest/) is a Python library for working with databases more easily/lazily. It's built on top of [SQLAlchemy](http://www.sqlalchemy.org). Features include Automatic Schemas, Upserts, Query helpers, Scripted exports and more.

Javascript
----------

* [Verlet.js](http://subprotocol.com/verlet-js/) is a simple Verlet integration physics engine.

* [AngularJS directives and the computer science of JavaScript](http://www.adobe.com/devnet/html5/articles/angularjs-directives-and-the-computer-science-of-javascript.html)

* [Getting started with AngularJS](http://www.adobe.com/devnet/html5/articles/getting-started-with-angularjs.html)

* [Angular UI](http://angular-ui.github.io) (is being broken into subprojects)

* [Superhero.js](http://superherojs.com) is a collection of the best articles, videos and presentations on the topic of maintaining a large Javascript codebase.

Geo
---

* [Soft Cities](http://softcities.net) prints [OpenStreetMap](http://www.openstreetmap.org) maps on your blankets! Awesome gift idea for geo nerds.

[![Image](/images/osm_blanket.jpg "OSM Blanket") ](http://softcities.net/item/Midnight-Commander-Blanket/1/c1)

* [Leaflet Data Visualization Framework](http://humangeo.github.io/leaflet-dvf/) is an extension to the Leaflet JavaScript mapping library with the primary goal to simplify data visualization and thematic mapping.

* [Finder using D3.Geo](http://dev.geosprocket.com/d3/finder/)

* [Geoscript in action part 1](http://blog.opengeo.org/2013/03/27/geoscript-in-action-part-one/)

* [GeoAlchemy 2](https://geoalchemy-2.readthedocs.org/en/0.1/) aims to be simpler than its predecessor, in terms of both usage, and maintainance.

* [postgis.net](http://postgis.net) - not sure how I missed this, but our beloved [PostgreSQL](http://www.postgresql.org) GIS extension has a new website.

* [ST_FromGDALRaster](http://postgis.net/docs/manual-dev/RT_ST_FromGDALRaster.html) does exactly that: Returns a raster from a supported GDAL raster file. But I didn't know it existed.

* [Infrared Band Analysis with Tilemill](http://mapbox.com/blog/infrared-band-analysis-tilemill/) makes use of [Mapbox.js](http://mapbox.com/mapbox.js/) to create this [sliding difference effect](http://mapbox.com/labs/ir/russia/#6.00/55.000/42.166)

* [Design Weather Maps](http://mapbox.com/blog/designing-weather-maps/), [Live Weather Data and MapBox.js](http://mapbox.com/blog/live-weather-with-mapboxjs/) and [Expanding TileMill's Building Symbolizer to Visualize Complex 3d Structures](http://mapbox.com/blog/tilemill-complex-3d-structures/) are more interesting reads from Mapbox.

* [OpenLayers 3 Animation Example](http://twpayne.github.io/ol3/full-screen-control/examples/animation.html) shows off the next generation of this [famous web mapping library](http://openlayers.org).

* [Displaying WebGL data on Google Maps](http://build-failed.blogspot.co.uk/2013/02/displaying-webgl-data-on-google-maps.html)

* [D3 + Leaflet](http://bost.ocks.org/mike/leaflet/)

* [WebGL GeoJSON tile rendering](http://bl.ocks.org/migurski/5024953)

* [Maps Shortcuts](http://www.youtube.com/watch?v=rrVT7Tp4yEg&feature=g-user-lss) is a Google Maps demonstration by Chris Broadfoot demonstrating an animated visualization of over 1 million GPS points from 25k cab rides in San Francisco.

* [MapD](http://data-informed.com/fast-database-emerges-from-mit-class-gpus-and-students-invention/) is a to-be-released GPU accelerated GIS database. Looking forward to that.

* [Streaming data via OPeNDAP](http://blog.dealmeida.net/streaming-data-via-opendap)

* [Pointillist](http://atogle.github.io/pointillist/paint.html) allows geospatial painting on a Leaflet canvas.

* [Learn OSM](http://learnosm.org/en/) learns you OpenStreetMap step-by-step.

* [HeyWhatsThat](http://www.heywhatsthat.com/layers.html)

* [Dark Sky Forecast API](https://developer.forecast.io/docs) [Raw data](http://forecast.io/raw/)

* [Using GDAL with Python: a basic introduction](http://blog.remotesensing.io/2013/03/using-gdal-with-python-basic-intro/)

* [Fulcrum App](http://fulcrumapp.com) is a generic but powerful mobile field data collection app.

Datavisualization
-----------------

* [Giant D3.js gallery](http://biovisualize.github.io/d3visualization/)

* [Threegraphs](http://threegraphs.com) is a [Three.js](http://mrdoob.github.io/three.js/)-powered charting library. Should be powerful stuff, considering the use of [Typed Arrays and WebWorkers](http://afshinm.github.io/50k/).

* [Venn Diagrams with D3.js](http://www.benfrederickson.com/2013/05/09/venn-diagrams-with-d3.js.html) takes the [Venn diagram](http://en.wikipedia.org/wiki/Venn_diagram) to the extreme. [Code on Github](https://github.com/benfred/venn.js)

* [Christchurch 2010 Timeline](http://bl.ocks.org/tnightingale/4718717)

* [How Selections Work](http://bost.ocks.org/mike/selection/) is another well executed and thus essential piece of D3.js documentation by [Mike Bostock](http://bost.ocks.org/mike/) himself. Must read if using D3. Also read [Nested Selections](http://bost.ocks.org/mike/nest/).

* [Infro](http://cakey.github.io/infro/) is a Javascript visualisation tool for anomalies and patterns in tabular data. [Repo](https://github.com/cakey/infro/).

* [Vega](http://trifacta.github.io/vega/) is a visualization grammar, a declarative format for creating, saving and sharing visualization designs.

* [Reusable Charting Patterns](http://dexvis.wordpress.com/2013/03/23/reusable-charting-patterns-part-3/)

* [Creating a data visualization tool using D3.js, Crossfilter and Leaflet](https://jeromegagnonvoyer.wordpress.com/2013/04/17/creating-a-data-visualization-tool-using-d3-js-crossfilter-and-leaflet-js/)

* [Pathfinding.js](https://github.com/qiao/PathFinding.js/) is a pathfinding library intended for games. See the [demo here](http://qiao.github.io/PathFinding.js/visual/). It implements the [A*](http://en.wikipedia.org/wiki/A*_search_algorithm), [Bidirectional](http://en.wikipedia.org/wiki/Bidirectional_search), [Breadth-first](http://en.wikipedia.org/wiki/Breadth-first_search), [Jump-point](http://harablog.wordpress.com/2011/09/07/jump-point-search/), [Best-first](http://en.wikipedia.org/wiki/Best-first_search) and the infamous [Dijkstra](http://en.wikipedia.org/wiki/Dijkstra%27s_algorithm) pathfinding algorithms. It's nice to play with them interactively, so I know what I'm doing when using them in [PGRouting](http://pgrouting.org).

[![Image](/images/pathfinding.jpg "Pathfinding") ](http://qiao.github.io/PathFinding.js/visual/)
