Neogeo & Dataviz
----------------
[![Image](/images/russia-left-behind.jpg "The Russia left behind") ](http://www.nytimes.com/newsgraphics/2013/10/13/russia/)

* [The Russia Left Behind](http://www.nytimes.com/newsgraphics/2013/10/13/russia/) tells the story of a journey using beautiful photography, interactive mapping and video. Excellent example of storytelling using 'new' media. Ofcourse produced by [Mike Bostock](http://bost.ocks.org/mike/) amongst others.

[![Image](/images/huizenprijzen_nos.jpg "NOS.nl datavisualisatie huizenprijzen") ](http://app.nos.nl/datavisualisatie/huizenprijzen-nederland/)

 * [Huizenprijzen Nederland](http://app.nos.nl/datavisualisatie/huizenprijzen-nederland/) by [NOS](http://www.nos.nl/). Nice to see our national television company create some useful data visualization. ([more](http://app.nos.nl/datavisualisatie/))

 * [Global Slavery Index](http://www.globalslaveryindex.org)

 * [How the Financial Times has improved its mapping capabilities](http://www.journalism.co.uk/news/how-the-financial-times-has-improved-its-mapping-capabilities/s2/a553781/) - [example](http://www.ft.com/cms/s/2/851bed28-7077-11e2-a2cf-00144feab49a.html#axzz2bTJZqjWV)

 * [Crosslet](http://sztanko.github.io/crosslet/) for interactive visualisation and analysis of geostatistical datasets.

 * [PySAL](http://pysal.readthedocs.org/) is a Python bases spatial analysis library which enables things like [this](http://nbviewer.ipython.org/urls/raw.github.com/pysal/notebooks/master/PySAL_io.ipynb).

 * [Datamaps](https://github.com/ericfischer/datamaps) indexes points and lines and generates map tiles to display them.

 * [ggplot for Python](https://github.com/yhat/ggplot/) by [Yhat](http://blog.yhathq.com/posts/ggplot-for-python.html) seems like a good candidate for building a quick Google Image charts clone.

 * [3D World Map](http://oi.qoop.io/fhs/clientside/#europe)

 * [F4 Maps](http://map.f4-group.com/#lat=52.0927100&lon=5.1155732&zoom=18&ui.showMenuPage=true&menuPage.type=search&menuPage.id=zakkendragershof%2C%20utrecht&camera.theta=0.9) renders 3D [OpenStreetMap](https://www.google.nl/search?client=safari&rls=en&q=www.openstreetmap.org&ie=UTF-8&oe=UTF-8&gws_rd=cr&ei=USBpUqG-J4eN0AXNz4GYCg) data in a beautiful way using WebGL.

 * [Modern OpenGL Guide](http://open.gl)

 * [OV Zoeker](http://ovzoeker.nl) visualizes public transportation in The Netherlands in near real time. [Data](https://www.ndovloket.nl)/[Author](https://github.com/Overv)

 * [Lumion3D](http://lumion3d.com/lumion-4-0-update/)

 * [Namahn](http://namahn.com)

 * [IBikeCPH](http://www.ibikecph.dk) or I Bike Copenhagen is the best example of [OSRM](http://project-osrm.org) aside of their official demo server. It uses [Lua](http://www.keplerproject.org/luasql/) bindings to query a PostGIS database for landuse properties to route cyclists away from industrial zones to improve their trips' quality. More on using external data in OSRM [at MapBox](https://www.mapbox.com/blog/osrm-using-external-data/).

 * [Good Ol' Excel Is The Ultimate Data Visualization Tool (In Most Cases)](http://insights.qunb.com/good-ol-excel-is-the-ultimate-data-visualization-tool-in-most-cases)

 * [GeoTrellis Travelshed demo](http://transit.geotrellis.com/travelshed.html). ([source](https://github.com/geotrellis/geotrellis-transit)). Did I mention that [GeoTrellis](http://geotrellis.github.io) very fast?

 * [Dark Sky: Open Source](http://journal.darkskyapp.com/2012/open-source/): [cache-helpers](https://github.com/darkskyapp/cache-helpers), [pngparse](https://github.com/darkskyapp/pngparse), [sphere-knn](https://github.com/darkskyapp/sphere-knn), [string-hash](https://github.com/darkskyapp/string-hash) and [tz-lookup](https://github.com/darkskyapp/tz-lookup)

 * [nx_spatial](https://bitbucket.org/gallipoli/nx_spatial/wiki/Home) is a spatial extension for [NetworkX](http://networkx.github.io).

 * [leaflet-geojson-stream](https://github.com/tmcw/leaflet-geojson-stream)


[![Image](/images/comp.png "d3.ForceBundle")](https://github.com/upphiminn/d3.ForceBundle)

 * [d3.ForceBundle](https://github.com/upphiminn/d3.ForceBundle)

 * [d3 Sequences Sunburst](http://bl.ocks.org/kerryrodden/7090426)

 
Machine Learning
----------------

  * [BigML](https://bigml.com/how_it_works) is ML as a service.

  * [PredictionIO](http://prediction.io/) is another ML as a service but freely available on [Github](https://github.com/PredictionIO/PredictionIO).

  * [Theoria.io](http://theoria.io) is another ML as a service but seems to have stalled/died. I like how it's based on the excellent [Vowpal Wabbit](https://github.com/JohnLangford/vowpal_wabbit). Anyone knows more about Theoria?

  * [NLTK: Analyzing the meaning of sentences](http://nltk.googlecode.com/svn/trunk/doc/book/ch10.html) is a chapter from the Natural Language Toolkit for Python. I've used it in an iPython Notebook (available if you're interested) to play with generating of SQL from natural language. 

  * [MAUI indexer](https://code.google.com/p/maui-indexer/) [demo](http://maui-indexer.appspot.com)

  * [Elastic Search OpenNLP Plugin](https://github.com/spinscale/elasticsearch-opennlp-plugin) (github) based on [Apache OpenNLP](http://opennlp.apache.org/documentation/1.5.3/manual/opennlp.html)

  * [Elastic Search Suggester Plugin](https://github.com/spinscale/elasticsearch-suggest-plugin) (github)

  * [Elastic Search River Streaming JSON](https://github.com/spinscale/elasticsearch-river-streaming-json) (github)
