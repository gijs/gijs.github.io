 * [Touchstone.js](http://touchstonejs.io) looks like just another Javascript UI framework, but it seems interesting because it's based on React.js. The [demo](http://demo.touchstonejs.io) works pretty smoothly.

 * [What is the Flux Application Architecture](https://medium.com/brigade-engineering/what-is-the-flux-application-architecture-b57ebca85b9e) offers a good overview and walkthrough of the Flux application architecture.

 * [Flux Step By Step](http://blogs.atlassian.com/2014/08/flux-architecture-step-by-step/) by Nicola Paolucci of [Atlassian](http://www.atlassian.com/).

 * [spoike/refluxjs](https://github.com/spoike/refluxjs) is a library for uni-directional dataflow application architecture inspired by ReactJS Flux.

 * [On React](http://www.jasimabasheer.com/posts/on-react.html): A React component is a self-contained unit that encapsulates both the view and view logic. This abstraction is fundamental to a front-end application, just like classes are to an object-oriented app, or tables to a database app.

 * [2 years with Angular](http://www.fse.guru/2-years-with-angular): Verdict: good enough (not really)

 * [Using React.js to draw dynamic SVGs](http://biesnecker.com/2014/10/22/using-reactjs-to-draw-dynamic-svgs/)

 * [From Backbone To React- Our Experience Scaling a Web Application](http://www.techsonian.net/2014/09/from-backbone-to-react-our-experience-scaling-a-web-application/)

 * [React Components](http://react-components.com)

 * [reactjs/react-magic](https://github.com/reactjs/react-magic)

 * [Yahoo! is moving to React](http://www.slideshare.net/rmsguhan/react-meetup-mailonreact)

 * [A React.js case study](http://blog.krawaller.se/posts/a-react-js-case-study/) and [its follow-up post](http://blog.krawaller.se/posts/a-react-js-case-study-follow-up/)

 * [From Angular.js to React - the isomorphic way](http://blog.risingstack.com/from-angularjs-to-react-the-isomorphic-way/)

 * [Community Round-up #23](http://facebook.github.io/react/blog/2014/10/17/community-roundup-23.html)
