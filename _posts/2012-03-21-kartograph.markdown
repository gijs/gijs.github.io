[Kartograph](http://kartograph.org/) is a framework by [Gregor Aisch](http://gka.me/) (check out [his showcase](http://driven-by-data.net/) and [his blog](http://vis4.net/blog/)) for building interactive map applications without Google Maps or any other mapping service. It was created with the needs of designers and data journalists in mind.

Check out the [Projections demo](http://kartograph.org/showcase/projections/#ortho) and the [Three-dimensional mapping demo](http://kartograph.org/showcase/3d/). 

More showcase [here](http://kartograph.org/showcase/).

The core concept of Kartograph is to separate the mapping process in the map generation and map rendering part. The client-side library [kartograph.js](http://kartograph.org/about/kartograph.js/) renders pre-generated maps stored in SVG files.

You can chose among hundrets of ready-to-use maps or simply generate your own using [kartograph.py](http://kartograph.org/about/kartograph.py/), the open source Python SVG map generator. [More about this](http://kartograph.org/about/).
