'Neogeography' & Data visualization
-----------------------------------

[![Image](/images/here_torque.jpg "Living Cities") ](http://here.com/livingcities/#cities/helsinki)

 * [Living Cities](http://here.com/livingcities/#cities/helsinki) is a fantastic combination of [CartoDB](http://cartodb.com), CartoDB's [Torque](http://cartodb.github.io/torque/examples/osm.html) and Nokia's [HERE](http://here.com). Top of the list!
 
 * [Syria conflict: a year of deaths mapped](http://www.guardian.co.uk/news/datablog/interactive/2012/aug/02/syria-deaths-map?fb=native) uses [Torque](http://cartodb.github.io/torque/examples/osm.html) as well.

 * [A city's heartbeat](http://xumx.me/geospatial/#)

 * [OpenStreetMap Data Report 2013](http://www.mapbox.com/osm-data-report/) looks awesome.

 * [Why Map Portals Don't Work](http://mapbrief.com/2013/02/05/why-map-portals-dont-work-part-i/) ([ii](http://mapbrief.com/2013/02/07/paralysis-of-choice-why-map-portals-dont-work-part-ii/), [iii](http://mapbrief.com/2013/02/11/the-tyranny-of-requirements-why-map-portals-dont-work-part-iii/), [iv](http://mapbrief.com/2013/02/19/an-iconography-of-confusion-why-map-portals-dont-work-part-iv/), [v](http://mapbrief.com/2013/02/21/the-waiting-is-the-hardest-part-why-map-portals-dont-work-part-v/))
 
 * [Visual Sedimentation](http://www.visualsedimentation.org) is a JavaScript library for visualizing streaming data, inspired by the process of physical [sedimentation](http://en.wikipedia.org/wiki/Sedimentation).
 
 * [Vector data representation for WebGL](https://github.com/openlayers/ol3/wiki/Vector-Data-Representation-for-WebGL)
 
 * [Voodoo.js](http://www.voodoojs.com) combines WebGL with regular HTML into a lively mix. Just scroll down to see it if your system supports it.

 * [Make maps better, together](http://ben.balter.com/make-maps-better-together/) by [Ben Balter](https://twitter.com/benbalter) of Github. ([another article worth reading](http://ben.balter.com/2013/07/02/a-brief-history-of-the-internet/))

 * [OpenData Globe](http://dev.citysdk.waag.org/visualisation/) by [Waag Society](http://waag.org/). ([more](http://dev.citysdk.waag.org))

 * [jQuery-Mapael](https://github.com/neveldo/jQuery-Mapael) ([example](http://jsfiddle.net/neveldo/TUYHN/embedded/result/), [example](http://jsfiddle.net/neveldo/n6XyQ/embedded/result/))

 * [Interactive visualization The Guardian](http://www.guardian.co.uk/world/interactive/2013/jul/02/guardian-guide-extradition-interactive) using D3.js and D3.Geo
 
 * [The Insanely Illustrated Guide to Your First Data-Driven Tilemill Map](http://dataforradicals.com/the-insanely-illustrated-guide-to-your-first-tile-mill-map/)

 * [Leaflet.Elevation](http://mrmufflon.github.io/Leaflet.Elevation/) is a Leaflet plugin to view interactive height profiles of polylines lines using d3.
 
 * [Leaflet.Sync](https://github.com/turban/Leaflet.Sync) synchronizes two maps.

 * [GrowRows](http://growrows.com/) is a great experiment built with [Coopy](http://share.find.coop) ([Github repo](https://github.com/paulfitz/coopy))
 
 * [Git (and Github) for data](http://blog.okfn.org/2013/07/02/git-and-github-for-data/)

 * [Google Correlate](http://www.google.com/trends/correlate/draw?p=us)

 * [RBush](https://github.com/mourner/rbush) is a JavaScript library for high-performance 2D spatial indexing of points and rectangles.

 * [uMap](http://umap.fluv.io/en/) lets you create maps with OpenStreetMap layers in a minute and embed them in your site. ([demo](http://umap.fluv.io/en/map/test-page_204#5/14.775/-17.402))

 * [GeoSense](http://map.safecast.org) is a publishing platform for geospatial data. ([Github](https://github.com/Safecast/GeoSense))

 * [gs2geojson](https://npmjs.org/package/gs2geojson) is a simple google spreadsheet to geoJSON converter for node.js.
 
 * [sheetsee.js](http://jlord.github.io/sheetsee.js/) - ([demo](http://jlord.github.io/hack-spots/))

 * [PostGIS 3D Demo](http://vimeo.com/54776907) by [Oslandia](http://www.oslandia.com) ([with slides](http://vimeo.com/54927052))
 
 * [GraphHopper](https://github.com/graphhopper/graphhopper) is a fast Java routing engine for roads and more. ([live](http://graphhopper.com))

 * [Andrew Hill - Using OpenStreetMap Data to Build Your Dynamic Map](http://vimeopro.com/openstreetmapus/state-of-the-map-us-2013/video/68096663)

 * [Dymo](https://github.com/wrobstory/Dymo) is a map label placer with simulated annealing.
 
 * [Creating and Publishing Maps with D3, Dymo, and PhantomJS](http://wrobstory.github.io/2013/06/creating-with-d3-dymo.html)
 
 * [Ogre](http://ogre.adc4gis.com) is an ogr2ogr web client (this rocks, who wants to build a RESTful interface to GDAL?)
 
 * [Fiona](http://toblerity.github.io/fiona/README.html#usage) is OGR's neat, nimble, no-nonsense API.
 
 * [Vincent](https://github.com/wrobstory/vincent) is a Python to [Vega](https://github.com/trifacta/vega) translator. 
 
 * [Vincent Spatial File conversion with Ogre](http://wrobstory.github.io/2013/05/vincent-spatial-ogre.html)
 
 * [Quandl and Pandas](http://wrobstory.github.io/2013/04/quandl-pandas.html)
 
 * [Quattro Shapes](http://quattroshapes.com)
 
 * [When Maps Shouldn’t Be Maps](http://www.ericson.net/content/2011/10/when-maps-shouldnt-be-maps/)
 
 * [UTM zones with D3.js](http://bl.ocks.org/turban/5866872)
 
 * [Color theory and mapping](http://source.mozillaopennews.org/en-US/learning/color-theory-mapping/)
 
 * [Datawrapper](http://datawrapper.de) is an open source tool helping anyone to create simple, correct and embeddable charts in minutes.
 
 * [Bearcart](https://github.com/wrobstory/bearcart) can create [Rickshaw.js](http://code.shutterstock.com/rickshaw/) visualizations with [Python Pandas](http://pandas.pydata.org).
  
 * [Folium](https://github.com/wrobstory/folium) is Python data + Leaflet maps.
 
 * [Another vector tiles example](http://bl.ocks.org/mbostock/5798874), and [another one](http://visuallybs.com/vector-osm/#16/39.8768/-122.3998), and [another](https://github.com/springmeyer/svg-tiles).

 * [Amsterdam Campus](http://tulpinteractive.com/amsterdam-campus/)
 
 * [Nanocubes](http://www.nanocubes.net/#software) are a fast datastructure for in-memory data cubes. [Example](http://www.nanocubes.net/view.html#twitter_new_york)
 
 * [Visualization of Time-Oriented Data](http://www.timeviz.net) seems like an interesting but overpriced book
 
 * [Rasdaman](http://www.rasdaman.org) is a multidimensional array server (like THREDDS) that I somehow missed. Has a [GDAL driver](http://www.gdal.org/frmt_rasdaman.html) as well.
 
 * [Developing a D3.js edge](https://gumroad.com/l/vyYr)
  
 * [Stockholm 2070](http://www.stockholm2070.se)

 * [More Beautiful Maps: Announcing DynamicMapsEngineLayer](http://googlegeodevelopers.blogspot.nl/2013/06/more-beautiful-maps-announcing.html)

 * [Using Open Source GIS tools for spatial data - QGIS, GDAL and Python](https://www.openshift.com/blogs/using-open-source-gis-tools-for-spatial-data-qgis-gdal-python) which talks about [shapefileconverter](https://github.com/thesteve0/shapefileconverter)

 * [AngularJS + D3.js = Radian](http://www.skybluetrades.net/blog/posts/2013/04/24/radian/index.html)

 * [Rdf ontology on drug discovery data](https://github.com/dbcls/bh13/wiki/Rdf-ontology-on-drug-discovery-data) - Learn and practice the-state-of-the-art linked open data and Semantic Web technologies for drug discovery.

 * [Weathertable](http://weathertableapp.com)

 * [Cartaro](http://cartaro.org) - uses Drupal..argh!

 * [Terraformer](https://github.com/Esri/Terraformer) by ESRI. Who said they don't do Open Source?
